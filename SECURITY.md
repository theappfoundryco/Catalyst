# Security Policy

## Supported versions

Catalyst auto-updates via Sparkle. Only the **latest released version** is supported — older
versions are deprecated on each release and receive no fixes. Update before reporting.

## Reporting a vulnerability

**Do not open a public issue for a security problem.**

Report privately, either:

- **GitHub** — the repository **Security** tab → **Report a vulnerability** (private advisory), or
- **Email** — security@theappfoundry.co

Please include the affected version, your macOS version, reproduction steps, and the impact you
observed. A proof of concept helps. We aim to acknowledge within a few days and to ship a fix in
the next release once confirmed; we'll credit you in the advisory unless you prefer otherwise.

## Scope — what we especially want to hear about

Catalyst runs with the user's privileges and touches sensitive parts of their system, so the
highest-value reports concern:

- **The update path** — anything that could deliver, or trick Sparkle into accepting, an update
  that isn't the one we signed (EdDSA signature bypass, feed/enclosure tampering, downgrade).
- **The privileged helper** — privilege escalation or unauthorized actions through it.
- **Shell / environment handling** — command or config injection via the managed `~/.zshrc`
  blocks, shortcuts, aliases, or any shelled-out command.
- **Credential handling** — SSH keys, admin credentials, or snapshot secrets being exposed,
  logged, or written outside the Keychain.

## Out of scope

Findings that require a compromised machine or physical access, issues only in unsupported
(deprecated) versions, and reports generated by automated scanners without a demonstrated impact.
